syntax-highlighted

The whole app is one continuous working session — a lit terminal, not a dashboard. No cards, no panels, no rounded rectangles: structure comes from hairlines, indentation, and type. Color does semantic work like an editor theme: amber is your voice, cyan is the machine's, violet is the coach's, and green/red speak only as verdicts. Dark-only, by commitment.

// 04 the rules

What this UI refuses to do

R1No cards

Structure comes from hairlines, indentation, and type scale. If a box appears, it must be a real container of state (a diff, a specimen) — never decoration.

R2Hue is meaning

Amber = you, cyan = machine, violet = coach, green/red = verdicts from real runs. Nothing is colored to look nice.

R3Stale things fade

Health carries through luminance. An untested artifact dims until re-verified; attention goes where the light is.

R4The session is the interface

Ask, receipt, coach, library — all read top-to-bottom as one working document. No modal graveyards, no widget grids.

Dark-only is a commitment, not an omission — this direction lives in one visual world. If a light variant is ever wanted, it's a second syntax theme ("daylight"), designed on its own terms. Annotations toggle: click "annotations on" above the specimen.
