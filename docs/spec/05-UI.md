## 3. Page: Ask (`/`) — the hero flow, state machine

States: `idle → profiling → interrogating → compiling → drafted → receipting → receipted → saved`.

1. **idle**: centered composer. A borderless `textarea` (autofocus) styled as display
   type: `font-family: var(--display); font-weight:700; font-size: clamp(22px,3.4vw,32px);
   color: var(--human); background: transparent; border: none; outline: none;` with
   placeholder `What do you need done?` in `--tx-faint`. Below it a hint line
   `Ask anything · the studio asks back before it builds` (13px, `--tx-faint`).
   Model select (native `<select>` restyled minimal, options from /api/models where
   `available`) + gold pill button `Build it`.
2. **profiling**: keep the ask on screen as static headline (same styles, no box);
   show `Thinking about what to ask you…` in `--tx-faint` with a soft pulsing opacity
   animation (disabled under `prefers-reduced-motion`).
3. **interrogating**: render questions one at a time, mockup dialogue style: question
   in `--tx-dim` 15px; answer input = borderless text input in `--human` 16px,
   indented 18px; Enter submits and reveals the next. After each answered question
   whose `expected_check ≠ none`, show the mint line: `✓ check minted: <label>` in
   `--pass` 13px with `— from your answer` in `--tx-faint`. Skippable: a `Skip` ghost
   link per question.
4. **compiling**: `Compiling for <model>…` status line.
5. **drafted**: render the board (mockup `.board`): left, the prompt sheet — cyan wash,
   `border-radius: 20px`, padding 30px 32px, `--mono` 13.5px/1.85, XML-ish tags colored
   `--machine`, `{{vars}}` colored `--human`; annotated excerpts get numbered violet
   pins (17px circles). Right, floating notes: violet glow dot (12px, box-shadow
   `0 0 12px rgba(199,146,234,.5)`), title in `--display` 16.5px `--coach`, note text
   13.5px `--tx-dim`, optional `Why for this model:` line 12.5px `--tx-faint`, and if
   the technique file exists, a link `card ›` that opens a side sheet rendering the
   technique markdown. Hover/focus on pin or note toggles class `lit` on both (wash +
   inset edge / dot scale + glow) — same interaction as the mockup.
   Under the board: run-meta row (model in `--machine`, `v1`, cost) + gold pill
   `Prove it (~$X)` + ghost pill `Save without proof`.
6. **receipting**: progress line `Running your ask vs the compiled prompt on <n>
   inputs…`.
7. **receipted**: the compare grid (mockup `.compare`): left `.was` fail-wash titled
   `Your ask, as written`, right `.now` pass-wash titled `The compiled prompt`; each
   shows the output for the selected input (input selector: `input 1/2/3` links) in
   `--mono` 12.5px `--tx-dim`, then per-check lines `✓/✕ <label>` colored pass/fail
   with short detail. Tally row: `Compiled X/Y · original Z/Y · N inputs` in `--pass`
   bold + `$C total`. Buttons: gold `Save to library`, ghost `Refine first`.
8. **saved**: toast `Saved <title> — v1` + link to `/library/<slug>`; reset to idle.

Errors at any step: inline line in `--fail`, never a modal; `budget_exceeded` message
verbatim from server.

## 5. Page: Library (`/library`)
Header `Library — <n> artifacts, sorted by attention` (13px uppercase `--tx-faint`).
Rows per mockup `.item`: status dot (glow), title 15.5px bold, kind 12px `--tx-faint`,
right-aligned stat (`X/Y · <relative time>`; regressed in `--fail`). Stale rows
`opacity: .42`. Row click → `/library/:slug`.

## 6. Page: Artifact detail (`/library/:slug`)
Title (display, 28px), status line, target model chip. Sections: the PromptSheet with
notes (same as Ask drafted); a `Run` block: one input field per variable (textarea,
borderless, machine-wash), instrument toggles `interpretation` / `consistency`
(checkboxes), gold `Run ($est)`; results render output + CheckLines + instrument panels
(07-INSTRUMENTS §3). A `Refine` block: feedback textarea + gold `Refine`. A `History`
list: `v3 · <date> · <changelog summary>`, click renders that version read-only.

## 7. Page: Models
Two top-level groups, then category sections within each:
- **Hosted** — sections by `category` (general / coding / reasoning / vision /
  utility), cards sorted by `last_reviewed` desc.
- **Local** — sections by `vram_class` (8gb first, then 4gb, then others), and
  within each tier grouped by category showing `local_rank` (¹ ² ³ medallions in
  `--machine`); a header line states the tier's hardware meaning ("fits alongside
  KV cache in 8 GB at Q4").
Each card block: label (display 18px), id in `--machine` mono, provider,
`card_depth: thin` badge in `--human` when thin, `last_reviewed` with staleness note
if > 120 days (`--tx-faint` → `reviewed 5 months ago` in `--fail` if > 180d),
prompting_notes as body text. P8 adds GaugeRows from `measured` (bar: 7px,
`rgba(216,221,230,.08)` track, cyan gradient fill with glow) and footer
`measured, not asserted`.

## 8. Accessibility & motion
All interactive elements keyboard reachable; pin/note cross-highlight fires on focus;
`prefers-reduced-motion` disables pulse/glow/tilt animations; aligned digits get
`font-variant-numeric: tabular-nums`; sheets with long payloads get `overflow-x: auto`.
